Deploying MyMiniCloud to AWS EC2

📷 Add your image here (recommended: 1280×560px)

Running MyMiniCloud locally is great for development, but the real test is deploying it to a cloud VM. This guide walks through the full process of launching an AWS EC2 instance and getting all 9 containers running in production.

Prerequisites

  • An AWS account with EC2 access
  • An SSH key pair (.pem file)
  • The MyMiniCloud repository pushed to GitHub

Step-by-Step Deployment

1

Launch EC2 Instance

Choose Ubuntu 22.04 LTS, instance type t2.small (2GB RAM recommended). In Security Groups, open ports: 22, 80, 8080, 8081, 8085, 9000, 9001, 9090, 3000.

2

SSH into the Instance

ssh -i your-key.pem ubuntu@<EC2-PUBLIC-IP>
3

Run the Deploy Script

The project includes a deploy.sh script that installs Docker, clones the repo, and starts all containers automatically.

curl -O https://raw.githubusercontent.com/YOUR_USERNAME/MyMiniCloud/main/deploy.sh
bash deploy.sh
4

Verify All Services Are Running

docker compose ps
# All services should show "Up"
5

Access the Platform

Once running, the services are accessible at:

  • Dashboard: http://<EC2-IP>:8080
  • App API: http://<EC2-IP>:8085/health
  • Auth: http://<EC2-IP>:8081/login
  • MinIO Console: http://<EC2-IP>:9001
  • Grafana: http://<EC2-IP>:3000
  • Prometheus: http://<EC2-IP>:9090
⚠️ RAM Warning: Running 9+ containers on t2.micro (1GB RAM) may cause OOM errors. Add a 2GB swap file:
sudo fallocate -l 2G /swapfile && sudo chmod 600 /swapfile && sudo mkswap /swapfile && sudo swapon /swapfile

Updating the Deployment

After pushing new code to GitHub, update the EC2 deployment with:

cd ~/MyMiniCloud
git pull
docker compose up --build -d
